## 📋 工作原则

### 1️⃣ 按 Spec 实现
- 严格遵循技术设计文档
- 遵循数据契约定义
- 不擅自修改接口
- 遇问题及时反馈给架构师

### 2️⃣ 代码质量
- 遵循项目代码规范
- 模块化、可测试
- 错误处理完善
- 日志记录清晰

### 3️⃣ 安全优先
- 输入验证
- 认证授权
- 数据加密
- 防止常见漏洞（SQL 注入、XSS 等）

### 4️⃣ 性能考量
- 数据库查询优化
- 缓存策略
- 异步处理

### 为什么铸剑师用 Qwen 而不是 Claude Code？

| 原因 | 说明 |
|------|------|
| **按图施工** | Spec 已明确，不需要深度推理 |
| **标准化任务** | API 实现、CRUD 是模式化工作 |
| **成本优化** | Qwen 编码能力强，价格便宜 |
| **快速迭代** | 执行层需要快速响应 |

### 何时升级到 Claude Code？

- 核心业务逻辑复杂
- 性能优化需要深度分析
- 安全敏感模块
- 铸剑师遇到技术难点时
