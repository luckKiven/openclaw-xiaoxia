## 🎯 模型配置

### 默认配置
| 角色 | 默认模型 | 说明 |
|------|---------|------|
| 诸葛亮 | qwen3.5-plus | 日常协调，便宜够用 |
| 墨子 | **Claude Code** | 架构设计，深度推理 |
| 小白 | kimi-k2.5 | 需求分析，长文本 |
| 巧匠 | qwen3-coder-next | 前端实现，快速 |
| 铸剑师 | qwen3-coder-plus | 后端实现，性价比高 |
| 探雷 | glm-5 | 测试验证 |

### Claude Code 使用场景（贵，省着用）

**✅ 必须用 Claude Code 的场景：**
- 复杂系统架构设计
- 核心技术难点攻关
- 重要代码审查（Code Review）
- Spec 文档最终审核

**❌ 不需要用 Claude Code 的场景：**
- 简单功能实现
- 日常需求分析
- 文档编写
- 单元测试编写

**💡 成本优化策略：**
```
架构层 (墨子) = Claude Code     # 核心价值，值得
执行层 (巧匠/铸剑师) = Qwen    # 按 Spec 施工，不需要 Claude
```

## 🚀 召唤命令

```bash
# 基础召唤
sessions_spawn model="qwen3-max/qwen3.5-plus" label="诸葛亮" task="任务描述"

# 持久化会话
sessions_spawn mode="session" model="qwen3-max/qwen3.5-plus" label="诸葛亮" task="长期任务"

# ACP 运行时（Claude Code）
sessions_spawn runtime="acp" agentId="claude-code" label="墨子" task="架构设计"
```

## ⚠️ 使用说明

1. **任务分发** - 诸葛亮负责任务分析和分发
2. **结果汇总** - 所有 Agent 的输出由诸葛亮汇总给用户
3. **冲突解决** - 当多个 Agent 意见不一致时，诸葛亮有最终决定权
4. **成本控制** - 复杂任务优先使用 Claude Code，简单任务使用 Qwen
